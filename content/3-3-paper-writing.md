## 섹션별 작성 가이드

<div class="tab-container">
  <div class="tab-buttons">
    <button class="tab-btn active" data-tab="tab-pw-abstract">Abstract</button>
    <button class="tab-btn" data-tab="tab-pw-intro">Introduction</button>
    <button class="tab-btn" data-tab="tab-pw-method">Methodology</button>
    <button class="tab-btn" data-tab="tab-pw-results">Results</button>
    <button class="tab-btn" data-tab="tab-pw-conclusion">Conclusion</button>
  </div>
  <div class="tab-content active" id="tab-pw-abstract">
    <h4>Abstract 작성법</h4>
    <p>[배경 1-2문장 → 목적 1문장 → 방법 2-3문장 → 주요 결과 2-3문장 → 의의 1문장]</p>
    <p>[전체 200-250 단어 이내. 마지막에 쓰지만 가장 많이 읽히는 부분.]</p>
  </div>
  <div class="tab-content" id="tab-pw-intro">
    <h4>Introduction 구조</h4>
    <p>[큰 그림(왜 중요한가) → 기존 연구 정리 → 한계점/Gap → 본 연구의 목적과 기여]</p>
    <p>[Funnel 구조: 넓은 배경에서 좁은 연구 질문으로]</p>
  </div>
  <div class="tab-content" id="tab-pw-method">
    <h4>Methodology 작성법</h4>
    <p>[재현 가능하도록 상세히. 모델, 파라미터, 데이터, 전처리, 평가 지표 모두 명시]</p>
    <p>[그림으로 전체 프레임워크를 보여주면 효과적]</p>
  </div>
  <div class="tab-content" id="tab-pw-results">
    <h4>Results &amp; Discussion</h4>
    <p>[결과를 나열만 하지 말고 해석. "왜 이런 결과가 나왔는지" 설명]</p>
    <p>[기존 연구와 비교. 예상과 다른 결과가 있다면 원인 분석]</p>
  </div>
  <div class="tab-content" id="tab-pw-conclusion">
    <h4>Conclusion 작성법</h4>
    <p>[연구 요약 → 주요 발견/기여 → 한계점 → 향후 연구 방향]</p>
    <p>[새로운 내용을 추가하지 않음. Introduction의 목적에 답하는 형태]</p>
  </div>
</div>

## 글쓰기 실전 팁

<div class="highlight-box warning">
  <span class="highlight-box-icon">⚠️</span>
  <div class="highlight-box-content">
    <p><strong>흔한 실수:</strong></p>
    <ul>
      <li>[한 문단에 여러 주제를 섞는 것 → 1문단 = 1아이디어]</li>
      <li>[수동태 남용 → 명확한 주어 사용]</li>
      <li>[그림/표 설명 없이 넘어가기 → 모든 Figure/Table은 본문에서 언급]</li>
      <li>[참고문헌 관리 안 하기 → 처음부터 Zotero 등으로 관리]</li>
    </ul>
