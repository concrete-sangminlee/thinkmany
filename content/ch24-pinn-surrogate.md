# 물리 기반 AI와 대리 모델

공학에는 두 가지 전통이 있다. 하나는 물리 법칙에 기반한 수학 모델이고, 다른 하나는 데이터에 기반한 통계 모델이다. 유한요소법(FEM)은 물리 법칙을 편미분방정식(PDE)으로 표현하고 이를 수치적으로 풀어서 구조물의 응력, 온도, 유속을 계산한다. 머신러닝은 입력-출력 데이터 쌍에서 패턴을 학습하여 예측 모델을 만든다. 두 접근법 모두 강력하지만, 각각 명확한 한계가 있다.

물리 기반 모델은 정확하지만 느리다. 복잡한 3D 형상의 유한요소 해석 한 번에 수시간이 걸릴 수 있다. 설계 최적화처럼 수백, 수천 번의 반복 계산이 필요한 경우, 물리 시뮬레이션만으로는 현실적 시간 내에 답을 구하기 어렵다. 데이터 기반 모델은 빠르지만 물리를 무시한다. 충분한 데이터가 있으면 놀라운 예측 정확도를 보이지만, 훈련 데이터 범위 밖에서는 물리적으로 말도 안 되는 예측을 할 수 있다. 음(-)의 강도, 에너지 보존을 위반하는 결과, 열역학 제2법칙에 어긋나는 온도 분포. 순수 데이터 기반 모델은 이런 비물리적 결과를 아무렇지 않게 내놓는다.

이 두 세계를 융합하는 것이 현대 공학 AI의 핵심 방향이다. 물리의 정확성과 AI의 속도를 동시에 확보하겠다는 것이다. 이 장에서는 그 대표적인 두 가지 접근법인 Physics-Informed Neural Networks(PINN)과 대리 모델(Surrogate Model)을 다룬다.

## 왜 물리+AI 융합인가

이 질문에 대한 답을 구체적인 예로 시작해보자.

한 대학원생이 열교환기의 최적 설계를 연구하고 있다고 하자. 열교환기의 형상(핀 개수, 핀 높이, 유로 폭)을 바꿔가며 열전달 성능을 최대화하고 싶다. 전산유체역학(CFD) 시뮬레이션 한 번에 2시간이 걸린다. 설계 변수가 5개이고, 각 변수를 10수준으로 탐색하면 10^5 = 100,000회의 시뮬레이션이 필요하다. 200,000시간, 약 23년. 불가능하다.

유전 알고리즘 같은 최적화 기법을 쓰더라도 수백~수천 회의 시뮬레이션이 필요하다. 1,000회만 잡아도 2,000시간, 약 83일. 박사과정 중 한 연구에만 3개월을 시뮬레이션에 투자하는 것은 비현실적이다.

여기서 두 가지 선택지가 생긴다. 첫째, CFD를 돌리되 훨씬 적은 횟수만 돌리고, 나머지는 근사 모델(대리 모델)로 대체한다. 50~100회의 CFD를 전략적으로 수행하고, 그 결과로 빠른 예측 모델을 만들어서 최적화에 활용한다. 둘째, 신경망에 물리 법칙을 직접 주입하여, 적은 데이터로도 물리적으로 타당한 예측을 하게 한다. 이것이 PINN이다.

두 접근법은 서로 다른 문제를 해결한다. 대리 모델은 "비싼 시뮬레이션을 싸게 대체하는 것"이 목표이고, PINN은 "물리 법칙을 직접 학습에 반영하는 것"이 목표다. 둘 다 물리+AI 융합이지만, 관점과 방법론이 다르다. 이 장에서 둘 다 깊이 있게 다룬다.

## PINN 원리

Physics-Informed Neural Networks(PINN)는 2019년 Raissi, Perdikaris, Karniadakis가 제안한 방법론이다. 핵심 아이디어는 놀라울 정도로 단순하다. 신경망의 손실 함수에 물리 법칙(지배방정식)을 추가하는 것이다.

일반적인 신경망은 데이터 손실만 최소화한다. 입력 x를 넣으면 출력 y_pred가 나오고, 실제 값 y_true와의 차이(MSE 등)를 줄이도록 학습한다. PINN은 여기에 물리 손실을 더한다. 신경망의 출력이 지배방정식을 만족하는지 확인하고, 만족하지 않으면 페널티를 부여하는 것이다.

이것이 가능한 이유는 자동 미분(automatic differentiation) 덕분이다. PyTorch나 TensorFlow 같은 프레임워크는 신경망의 출력을 입력에 대해 자동으로 미분할 수 있다. 신경망이 u(x, t)를 출력하면, du/dx, du/dt, d^2u/dx^2 등을 자동으로 계산할 수 있다. 이 미분값들을 지배방정식에 대입하면, 잔차(residual)를 계산할 수 있다. 잔차가 0이면 지배방정식을 정확히 만족하는 것이고, 0이 아니면 물리 법칙을 위반하는 것이다.

**1단계: 지배방정식 정의.** 풀고자 하는 물리 문제의 지배방정식을 명확하게 정의한다. 예를 들어 1D 비정상 열전도 문제라면, du/dt = alpha * d^2u/dx^2 이다. 여기서 u는 온도, alpha는 열확산계수, x는 위치, t는 시간이다. 경계조건(예: u(0,t) = T_left, u(L,t) = T_right)과 초기조건(예: u(x,0) = f(x))도 함께 정의한다.

**2단계: 신경망 구성.** 보통 MLP(다층 퍼셉트론)를 사용한다. 입력은 공간 좌표와 시간(x, t)이고, 출력은 물리량(u)이다. 은닉층은 3~6개, 각 층에 20~50개의 노드를 사용하는 것이 일반적이다. 활성화 함수로는 tanh가 PINN에서 자주 쓰인다. ReLU는 2차 미분이 0이 되어 PDE 잔차 계산에 문제가 생길 수 있기 때문이다.

**3단계: 손실함수 설계.** 이것이 PINN의 핵심이다. 손실함수는 세 가지(또는 그 이상) 항의 가중합으로 구성된다. 이 부분은 바로 다음 절에서 자세히 다룬다.

**4단계: 학습.** 일반적으로 Adam 옵티마이저로 초기 학습을 진행한 후, L-BFGS로 미세 조정하는 2단계 전략을 사용한다. Adam은 초기에 빠르게 좋은 영역으로 수렴하지만, 최종 정밀도는 L-BFGS가 더 높다. 학습률은 1e-3에서 시작하여 점진적으로 줄이는 것이 보통이다.

**5단계: 예측과 검증.** 학습이 끝난 신경망에 임의의 (x, t) 좌표를 넣으면 해당 점에서의 물리량 u를 예측한다. 해석해가 있는 문제에서는 해석해와 비교하여 정확도를 확인하고, 실험 데이터가 있으면 실험과 비교한다.

## 손실 함수 설계

PINN의 손실 함수는 다음과 같은 구조를 가진다.

L = lambda_data * L_data + lambda_pde * L_pde + lambda_bc * L_bc + lambda_ic * L_ic

각 항을 살펴보자.

**L_data (데이터 손실).** 실험이나 시뮬레이션에서 얻은 관측 데이터와 신경망 예측의 차이다. N_data개의 데이터 포인트에서의 MSE로 계산한다. 관측 데이터가 풍부하면 이 항의 비중이 커지고, 관측 데이터가 없으면(순수 forward problem) 이 항은 0이 된다.

**L_pde (물리 손실 / PDE 잔차 손실).** 도메인 내부의 콜로케이션 포인트(collocation points)에서 지배방정식의 잔차를 계산한다. 열전도 방정식이라면, r(x,t) = du/dt - alpha * d^2u/dx^2 을 계산하고, 이 잔차의 MSE를 구한다. 잔차가 0에 가까울수록 신경망이 물리 법칙을 잘 만족하는 것이다. 콜로케이션 포인트는 보통 도메인 내에서 균일하게 또는 Latin Hypercube Sampling(LHS)으로 생성한다. 포인트 수는 수천~수만 개가 일반적이다.

**L_bc (경계조건 손실).** 경계에서의 조건 만족 여부를 측정한다. 디리클레 경계조건(u = 값)이면 경계 포인트에서 신경망 출력과 지정값의 차이를, 노이만 경계조건(du/dn = 값)이면 법선 방향 미분과 지정값의 차이를 계산한다.

**L_ic (초기조건 손실).** 시간 의존 문제에서 t=0일 때의 초기조건 만족 여부를 측정한다. L_bc와 마찬가지로 MSE로 계산한다.

### 가중치 조절의 어려움

네 개의 손실 항이 있으면, 각 항의 가중치(lambda)를 어떻게 설정하느냐가 핵심 문제가 된다. 이것이 PINN에서 가장 까다로운 부분 중 하나다.

가중치를 모두 1로 설정하면 어떻게 될까? 대부분의 경우 잘 작동하지 않는다. 각 손실 항의 크기(scale)가 다르기 때문이다. L_pde에 2차 미분이 포함되면 값의 크기가 수백~수천이 될 수 있고, L_data는 0~1 범위일 수 있다. 크기가 큰 항이 학습을 지배하면서 다른 항은 무시된다.

이 문제를 해결하기 위한 여러 전략이 있다.

**수동 조절.** 가장 단순하지만 시간이 오래 걸린다. 각 lambda를 시행착오로 조정한다. 보통 lambda_pde를 기준(1.0)으로 두고, 나머지를 0.001~1000 범위에서 조절한다. 경계조건 위반이 심하면 lambda_bc를 올리고, 데이터 적합이 안 되면 lambda_data를 올리는 식이다.

**그래디언트 노름 기반 자동 가중치.** Wang et al.(2021)이 제안한 방법이다. 각 손실 항이 신경망 파라미터에 미치는 그래디언트의 크기(norm)를 계산하고, 그래디언트 크기가 비슷해지도록 가중치를 자동으로 조절한다. 그래디언트가 큰 항의 가중치를 줄이고, 작은 항의 가중치를 키우는 것이다.

**Neural Tangent Kernel(NTK) 기반 가중치.** 학습 동역학 이론에 기반한 더 정교한 방법이다. 각 손실 항의 수렴 속도를 분석하여 가중치를 조절한다. 구현이 복잡하지만 이론적 근거가 탄탄하다.

## PINN 적용 분야

PINN은 지배방정식이 알려진 물리 문제에 폭넓게 적용할 수 있다. 현재 활발히 연구되고 있는 주요 분야를 살펴보자.

### 열전달

열전도 방정식은 PINN의 가장 기본적인 적용 대상이다. 정상 상태와 비정상 상태 모두 다룰 수 있다. 복잡한 형상(예: 전자 부품의 방열판)에서의 온도 분포를 예측하거나, 표면 온도 측정값으로부터 내부 온도 분포를 역추정하는 역문제에 활용된다. 특히 역문제에서 PINN이 강력하다. 기존 방법으로는 열원의 위치와 강도를 역추정하는 것이 매우 어렵지만, PINN은 관측 데이터와 물리 법칙을 동시에 만족하는 해를 직접 찾는다.

### 유체역학

나비에-스토크스 방정식에 기반한 유동 해석이 대표적이다. 저속 비압축성 유동에서의 속도장과 압력장을 PINN으로 예측한 연구들이 많다. 실험적으로 속도장을 부분적으로 측정한 후(PIV 데이터), PINN으로 전체 유동장과 압력장을 복원하는 연구도 활발하다. 압력은 직접 측정이 어렵기 때문에, 속도 데이터와 나비에-스토크스 방정식으로부터 압력을 추정하는 PINN 접근법은 실용적 가치가 높다.

### 고체역학

탄성체의 응력-변형률 관계, 판/쉘 구조물의 변형, 파괴역학에서의 응력 확대 계수 계산 등에 PINN이 적용되고 있다. 특히 복합재료와 같이 비균질 재료의 물성 분포를 역추정하는 문제에서 PINN이 주목받고 있다. 표면에서의 변위 측정값으로부터 내부의 탄성계수 분포를 추정하는 것은 전통적인 역문제 해석으로는 매우 어렵지만, PINN은 비교적 직관적인 접근을 가능하게 한다.

### 역문제(Inverse Problem)

역문제야말로 PINN이 가장 빛나는 영역이다. 역문제란, 관측 결과로부터 원인(물성치, 경계조건, 소스 항 등)을 추정하는 문제다. 예를 들어, 구조물 표면의 온도를 측정하여 내부 열원의 위치를 찾는 것, 구조물의 동적 응답을 측정하여 재료 물성치를 추정하는 것, 유동장 측정값으로부터 유체의 점성을 추정하는 것 등이다.

전통적인 역문제 해석은 반복적 최적화가 필요하고, 매 반복마다 순방향 해석(FEM, CFD)을 수행해야 하므로 계산 비용이 매우 높다. PINN은 미지의 물성치를 신경망의 학습 가능한 파라미터로 추가하여, 한 번의 학습 과정에서 물리량과 물성치를 동시에 구할 수 있다. 이것이 PINN의 가장 혁신적인 특징이다.

## PINN 예제: 1D 열전도

구체적인 예제를 통해 PINN의 작동 원리를 확인해보자.

### 문제 정의

1차원 정상 상태 열전도 문제를 생각하자. 길이 L=1인 막대의 양끝 온도가 고정되어 있고, 내부에 열원이 있다.

지배방정식: d^2u/dx^2 + f(x) = 0, 0 < x < 1

경계조건: u(0) = 0, u(1) = 0

열원: f(x) = sin(pi*x)

이 문제의 해석해는 u(x) = sin(pi*x) / pi^2 이다. 해석해가 있으므로 PINN의 정확도를 정밀하게 검증할 수 있다.

### 신경망 구조

입력: x (1개), 출력: u (1개). 은닉층 3개, 각 층에 20개의 노드. 활성화 함수: tanh. 전체 파라미터 수는 약 900개다. 이 정도 크기의 문제에서는 매우 작은 신경망이면 충분하다.

### 손실 함수

이 문제에서는 관측 데이터 없이 물리 법칙만으로 해를 구하므로 L_data = 0이다.

L = lambda_pde * L_pde + lambda_bc * L_bc

L_pde: 도메인 내부에서 100개의 콜로케이션 포인트를 균일하게 생성한다. 각 포인트에서 d^2u_nn/dx^2 + sin(pi*x) 의 잔차를 계산하고 MSE를 구한다.

L_bc: 경계 포인트 x=0과 x=1에서 u_nn(0)^2 + u_nn(1)^2 을 계산한다.

lambda_pde = 1.0, lambda_bc = 10.0 으로 설정한다. 경계조건은 반드시 만족해야 하므로 가중치를 높게 준다.

### 학습

Adam 옵티마이저, 학습률 1e-3, 10,000 에포크 학습 후 L-BFGS로 5,000 에포크 추가 학습. 학습 과정에서 L_pde와 L_bc를 모니터링한다. 둘 다 1e-6 이하로 떨어지면 수렴했다고 판단한다.

### 결과 비교

학습된 신경망의 예측값과 해석해를 비교하면, L2 상대오차가 약 1e-4 ~ 1e-5 수준으로 매우 정확하다. 이 단순한 예제에서는 PINN이 해석해를 거의 완벽하게 재현한다. 다만 문제가 복잡해지면(다차원, 비선형, 급격한 변화 등) 수렴이 어려워질 수 있다.

<div class="highlight-box warning">
<span class="highlight-box-icon">⚠️</span>
<div class="highlight-box-content">
<p><strong>PINN 수렴의 현실</strong></p>
<p>위 예제처럼 단순한 1D 문제에서는 PINN이 잘 작동한다. 그러나 2D/3D 문제, 비선형 PDE, 급격한 경계층(boundary layer), 다중 스케일 문제에서는 수렴이 매우 어려울 수 있다. 논문에서 "PINN으로 풀었다"고 나와 있어도, 실제 재현하면 수렴하지 않는 경우가 적지 않다. PINN은 강력한 도구이지만, 마법은 아니다.</p>
</div>
</div>

## PINN vs FEM

PINN이 기존 수치해석(FEM, FDM, FVM)을 대체할 수 있을까? 솔직한 답은 "아직 아니다"이다. 각 방법이 유리한 영역이 다르다.

<div class="do-dont">
<div class="do-box">
<h4>✅ PINN이 유리한 경우</h4>
<ul>
<li>역문제: 관측 데이터로부터 물성치나 경계조건을 추정할 때</li>
<li>희소 데이터 통합: 부분적 실험 데이터 + 물리 법칙으로 전체 해를 복원할 때</li>
<li>복잡한 형상의 메쉬 없는 해석: 메쉬 생성이 어려운 형상에서 (PINN은 메쉬가 필요 없다)</li>
<li>다중 물리 문제: 여러 물리 현상이 결합된 문제에서 유연한 적용 가능</li>
<li>파라미터 스터디: 경계조건이나 물성치를 바꿔가며 빠르게 근사해를 구할 때</li>
</ul>
</div>
<div class="dont-box">
<h4>❌ FEM/전통 수치해석이 유리한 경우</h4>
<ul>
<li>잘 정의된 순방향 문제: 물성치, 경계조건이 모두 알려진 표준 문제</li>
<li>높은 정확도가 필요한 경우: 메쉬 세분화(h-refinement)로 원하는 정확도 보장 가능</li>
<li>대규모 산업 문제: 수백만 자유도의 구조 해석, 상용 소프트웨어의 성숙도</li>
<li>오류 추정과 수렴 보장: FEM은 수학적 수렴 이론이 확립되어 있다</li>
<li>규격/인증: 산업 규격에서 인정하는 해석 방법</li>
</ul>
</div>
</div>

현재 PINN의 가장 실용적인 위치는 "FEM을 보완하는 도구"다. FEM으로 해결하기 어려운 역문제, 데이터 동화(data assimilation), 실시간 예측 등에서 PINN이 빛을 발한다. FEM을 완전히 대체하려는 시도보다는, FEM과 PINN을 결합하는 하이브리드 접근이 더 현실적이다.

## 대리 모델링

이제 대리 모델(Surrogate Model)로 넘어가자. 대리 모델은 PINN과는 다른 관점에서 물리와 AI를 연결한다.

### 왜 대리 모델이 필요한가

대리 모델의 동기는 단순하다. 고충실도(high-fidelity) 시뮬레이션은 정확하지만 느리다. 유한요소 해석 한 번에 30분~수 시간이 걸린다면, 최적화, 불확실성 분석, 민감도 분석처럼 수백~수천 번의 반복 계산이 필요한 작업을 직접 수행하는 것은 불가능하다.

대리 모델은 "비싼 함수를 싼 함수로 근사하는 것"이다. 제한된 횟수의 시뮬레이션(훈련 데이터)을 수행하고, 그 입출력 관계를 빠른 수학적 모델로 학습한다. 학습된 대리 모델은 밀리초 단위로 예측하므로, 최적화나 불확실성 분석에 직접 활용할 수 있다.

구체적인 예를 들어보자. 자동차 충돌 시뮬레이션(LS-DYNA)은 한 번에 6~12시간이 걸린다. 차체 두께 10개 부위를 설계 변수로 두고, 충돌 안전성을 만족하면서 중량을 최소화하려면 수백 번의 시뮬레이션이 필요하다. 100회의 시뮬레이션으로 대리 모델을 구축하면, 이후 최적화 알고리즘이 대리 모델을 수만 번 호출하여 최적해를 탐색할 수 있다. 총 계산 시간이 수개월에서 며칠로 줄어든다.

### 대리 모델링의 주요 기법

대리 모델에는 여러 종류가 있다. 각각의 특징과 적합한 상황이 다르다.

**반응 표면 방법론(Response Surface Methodology, RSM).** 가장 고전적인 대리 모델이다. 입출력 관계를 2차 다항식(quadratic polynomial)으로 근사한다. y = b0 + sum(bi*xi) + sum(bii*xi^2) + sum(bij*xi*xj). 계수(b)는 최소자승법으로 구한다. 변수가 적고(5개 이하) 관계가 비교적 단순한 경우에 효과적이다. 계수의 통계적 유의성을 검정할 수 있어서 변수 영향 분석에도 유용하다. 한계는 비선형성이 강한 관계를 포착하기 어렵다는 것이다.

**크리깅/가우시안 프로세스(Kriging/Gaussian Process, GP).** 대리 모델의 핵심이라 할 수 있다. GP는 함수에 대한 확률적 모델이다. 예측값뿐 아니라 예측의 불확실성(분산)도 함께 제공한다. 데이터가 밀집된 영역에서는 불확실성이 작고, 데이터가 없는 영역에서는 불확실성이 크다. 이 불확실성 정보는 베이지안 최적화에서 핵심적으로 활용된다.

GP의 핵심 요소는 커널(kernel) 함수다. Matern 커널, RBF(Squared Exponential) 커널, 주기 커널 등이 있으며, 커널의 선택이 모델 성능에 큰 영향을 미친다. 커널의 하이퍼파라미터(길이 스케일, 분산 등)는 최대 우도(maximum likelihood)로 추정한다. 단점은 데이터 수가 늘어나면 계산 비용이 O(N^3)으로 급격히 증가한다는 것이다. 일반적으로 수천 개 이상의 데이터에서는 비효율적이다.

**방사 기저 함수(Radial Basis Function, RBF).** 데이터 포인트를 중심으로 방사 함수(가우시안, 멀티쿼드릭 등)를 배치하고, 이들의 가중합으로 함수를 근사한다. 구현이 간단하고 보간(interpolation)이 정확하다. GP보다 계산이 가볍지만, 불확실성 추정이 어렵다.

**신경망(Neural Network).** 대규모 데이터와 고차원 입력에서 강력하다. 수십~수백 개의 입력 변수를 처리할 수 있고, 수만 개의 데이터를 효율적으로 학습한다. 다만 소규모 데이터(수십~수백 개)에서는 과적합 위험이 있고, GP처럼 자연스러운 불확실성 추정을 제공하지 않는다. 앙상블이나 MC Dropout 같은 기법으로 불확실성을 추정할 수 있지만 추가 작업이 필요하다.

**변수가 5개 이하이고 데이터가 수십 개 수준이라면, RSM이나 GP를 추천한다.**

RSM은 2차 다항식으로 근사하므로 물리적 해석이 쉽고, 변수의 주효과(main effect)와 교호작용(interaction)을 명확히 볼 수 있다. GP는 더 유연한 비선형 근사가 가능하고, 불확실성도 제공한다. 변수가 2~3개이면 RSM만으로 충분한 경우가 많다. 4~5개이면 GP가 더 안전한 선택이다.

실험 설계(DOE)와 함께 사용한다. 중심합성설계(CCD)나 Box-Behnken 설계로 효율적으로 시뮬레이션 포인트를 배치한다.

</div>

<div class="tab-content" id="tab-ch24-uncertainty">

**예측의 신뢰도(불확실성)가 중요하다면, GP가 최선이다.**

GP는 예측값과 함께 예측 분산을 제공한다. "이 영역은 데이터가 부족해서 예측이 불확실하다"는 정보를 정량적으로 알 수 있다. 이 불확실성은 베이지안 최적화, 신뢰성 분석, 로버스트 설계 등에서 핵심적으로 활용된다.

대안으로 신경망 앙상블(여러 신경망의 예측 분산)이나 MC Dropout도 불확실성을 추정할 수 있지만, GP의 불확실성이 이론적으로 더 정교하다.

</div>

<div class="tab-content" id="tab-ch24-large-scale">

**데이터가 수천 개 이상이라면, 신경망이 효율적이다.**

GP는 N^3 계산 비용 때문에 수천 개 이상의 데이터를 다루기 어렵다(Sparse GP로 완화할 수 있지만 근사 오차가 생긴다). 신경망은 미니배치 학습으로 대규모 데이터를 효율적으로 처리한다. XGBoost나 LightGBM 같은 트리 기반 모델도 대규모 정형 데이터에서 강력한 대안이다.

</div>

<div class="tab-content" id="tab-ch24-high-dim">

**입력 변수가 수십 개 이상이라면, 신경망이나 트리 기반 모델을 고려한다.**

GP와 RSM은 고차원에서 "차원의 저주"를 심하게 받는다. 20개 이상의 입력 변수가 있으면, 신경망이나 XGBoost가 더 실용적이다. 다만, 고차원 문제에서는 충분한 훈련 데이터가 필요하다. 변수당 최소 10~20개의 데이터 포인트가 일반적인 경험 법칙이다.

필요에 따라 차원 축소(PCA, 민감도 분석)를 먼저 수행하여 중요한 변수를 선별한 후, 축소된 변수 공간에서 GP를 적용하는 전략도 효과적이다.

### 초기 샘플링

설계 공간을 균일하게 탐색하기 위해 Latin Hypercube Sampling(LHS)을 사용한다. LHS는 각 변수의 범위를 N등분하고, 각 구간에서 정확히 하나의 점을 추출하여 공간을 효율적으로 채운다. 변수가 d개이면, 초기 샘플 수는 보통 5d~10d 정도가 적절하다. 변수가 5개이면 25~50개의 초기 점으로 시작한다.

### 획득 함수(Acquisition Function)

BO의 핵심은 "다음에 어디를 평가할 것인가"를 결정하는 획득 함수(acquisition function)다. GP가 예측 평균(mu)과 예측 불확실성(sigma)을 제공하므로, 이 두 정보를 조합하여 유망한 지점을 선택한다.

**Expected Improvement(EI).** 현재까지의 최적값보다 얼마나 개선될 것으로 기대되는가를 계산한다. EI가 높은 지점은 "현재보다 나을 가능성이 높은 곳"이다. 예측 평균이 좋거나(exploitation), 예측 불확실성이 큰(exploration) 지점에서 EI가 높다. 탐색(exploration)과 활용(exploitation)을 자동으로 균형 잡는다는 것이 EI의 강점이다.

**Lower Confidence Bound(LCB).** mu - kappa * sigma 를 최소화하는 지점을 선택한다. kappa가 크면 불확실성이 큰 곳을 우선 탐색하고(exploration), 작으면 예측값이 좋은 곳을 우선 활용한다(exploitation). kappa를 조절하여 탐색-활용 균형을 수동으로 제어할 수 있다.

### 반복과 수렴

획득 함수를 최적화하여 다음 평가 포인트를 선택하고, 그 포인트에서 실제 시뮬레이션을 수행하고, 결과를 GP에 추가하여 재학습하고, 다시 획득 함수를 최적화하는 과정을 반복한다. 보통 20~100회의 반복이면 수렴한다. 총 시뮬레이션 횟수가 초기 샘플 + 반복 횟수이므로, 50~150회의 시뮬레이션으로 최적화를 완료할 수 있다.

## 한계와 현실

PINN과 대리 모델은 강력한 도구이지만, 만능이 아니다. 연구에 적용하기 전에 알아야 할 현실적 한계들을 짚어보자.

### PINN의 한계

**수렴 어려움.** 앞서 언급한 바 있지만, 가장 빈번하게 마주치는 벽이므로 다시 짚는다. 급격한 경계층, 충격파(shock wave), 다중 스케일 현상, 높은 레이놀즈 수의 난류 유동 등에서 PINN은 고전한다. 수렴하지 않는 PINN을 수주간 디버깅하다가 결국 FEM으로 돌아가는 경우도 흔하다.

**계산 비용의 오해.** "PINN은 메쉬가 필요 없으므로 빠르다"는 오해가 있다. 학습 과정 자체가 수시간~수일이 걸릴 수 있고, 새로운 경계조건이나 형상에서는 처음부터 다시 학습해야 한다. FEM은 메쉬만 바꾸면 되지만, PINN은 재학습이 필요하다. 다양한 조건에서 반복적으로 해를 구해야 하는 경우에는 FEM이 더 효율적일 수 있다.

**이론적 보장 부족.** FEM은 수렴 속도, 오차 한계 등의 수학적 보장이 있다. PINN은 이런 이론적 보장이 아직 부족하다. "학습이 잘 되면 좋은 해를 주지만, 잘 될지 미리 알기 어렵다"는 것이 현재 상황이다.

### 대리 모델의 한계

**정확도 한계.** 대리 모델은 근사이므로, 원래 시뮬레이션과 정확히 같은 결과를 주지 않는다. 대리 모델의 오차가 허용 범위 내인지 반드시 검증해야 한다. 검증 데이터(훈련에 사용하지 않은 시뮬레이션 결과)로 R^2, RMSE, 최대 오차 등을 확인한다. R^2가 0.95 이상이면 대체로 쓸 만하나, 문제에 따라 더 높은 정확도가 필요할 수 있다.

**외삽의 위험.** 대리 모델은 훈련 데이터 범위 내에서만 신뢰할 수 있다. 범위 밖으로 외삽하면 예측이 급격히 나빠진다. 최적화 과정에서 대리 모델이 훈련 범위 밖의 영역을 "최적"이라고 제안하면, 그 결과는 신뢰할 수 없다. 제약 조건으로 탐색 범위를 제한하거나, 외삽 경고를 추가하는 것이 필요하다.

**차원의 저주.** 입력 변수가 많아지면, 설계 공간을 적절히 채우기 위해 필요한 훈련 데이터 수가 기하급수적으로 증가한다. 변수가 20개 이상이면, 합리적인 수의 시뮬레이션으로는 대리 모델의 정확도를 확보하기 어려울 수 있다. 차원 축소나 변수 선별이 사전에 필요하다.

### 연구자의 판단이 핵심이다

결국 PINN을 쓸지, 대리 모델을 쓸지, FEM을 쓸지, 아니면 조합할지는 연구자의 판단이다. 도구 자체보다 중요한 것은 "이 도구가 내 문제에 적합한가?"를 판단하는 능력이다.

몇 가지 실전 가이드라인을 제안한다.

첫째, 단순한 문제부터 시작한다. PINN을 처음 적용한다면, 해석해가 있는 1D/2D 문제부터 검증한다. 대리 모델을 처음 만든다면, 변수 2~3개의 단순한 문제부터 시작한다. 복잡한 문제에 바로 적용하면 문제가 도구의 한계인지, 내 구현의 버그인지 구분하기 어렵다.

둘째, 기존 방법과 반드시 비교한다. PINN으로 해석해가 있는 문제를 풀어서 정확도를 확인한다. 대리 모델은 검증 데이터로 예측 정확도를 확인한다. "새로운 방법을 썼다"는 것 자체는 논문의 기여가 되지 않는다. "기존 방법 대비 어떤 이점이 있는가"를 보여야 한다.

셋째, 물리적 타당성을 항상 확인한다. PINN이나 대리 모델의 예측 결과가 물리적으로 말이 되는지 확인한다. 온도가 음수인가? 응력이 비현실적으로 큰가? 에너지가 보존되는가? 수치적으로 좋아 보여도 물리적으로 틀리면 의미가 없다.

## 마무리: 물리와 AI 사이에서

물리 기반 AI와 대리 모델은 공학 연구에서 점점 더 중요해지고 있다. PINN은 물리 법칙을 신경망에 직접 주입하여 데이터가 부족해도 물리적으로 타당한 예측을 가능하게 하고, 역문제 해석에서 혁신적인 접근을 제공한다. 대리 모델은 비싼 시뮬레이션을 빠른 근사 모델로 대체하여 최적화와 불확실성 분석을 실현 가능하게 한다.

그렇다고 이 도구들이 기존 방법을 완전히 대체하는 것은 아니다. FEM은 여전히 산업 표준이고, 실험은 여전히 최종 검증의 기준이다. 새로운 도구의 가치는 "기존에 불가능했던 것을 가능하게 하는 것"이다. PINN으로 역문제를 풀어서 비파괴 검사의 정확도를 높이고, 대리 모델로 설계 공간을 효율적으로 탐색하여 최적 설계를 찾는 것. 이런 구체적인 기여가 논문이 된다.

물리를 이해하는 공학자가 AI를 도구로 활용할 때, 가장 강력한 연구가 나온다. 물리를 모르는 AI 연구자는 비물리적 결과를 걸러내지 못하고, AI를 모르는 공학자는 새로운 도구의 가능성을 놓친다. 두 영역을 잇는 위치에 서는 것이 공학 대학원생의 전략적 선택이다.

## PINN의 실전 함정과 디버깅

ch24 본문에서 PINN의 원리와 가능성을 다뤘다. 하지만 실제로 PINN을 학습시켜 본 사람은 안다. PINN은 종이 위에서는 우아하지만 실전에서는 까다롭다. 학습이 수렴하지 않거나, 수렴해도 물리적으로 잘못된 답을 내거나, 아주 단순한 문제는 잘 되지만 조금만 복잡해지면 실패한다. 본 섹션은 PINN을 실제로 사용할 때 만나는 함정과 해결책을 정리한다.

<div class="highlight-box warning">
  <span class="highlight-box-icon">⚠️</span>
  <div class="highlight-box-content">
    <p><strong>"논문에서는 잘 되는데 내 문제에서는 안 된다"의 진실</strong></p>
    <p>PINN 논문들은 대부분 1D, 2D의 매끄러운 PDE에서 인상적인 결과를 보여준다. 본인이 풀고 싶은 3D, 비선형, 경계층, 충격파, 다물리 문제는 훨씬 어렵다. 이것은 본인의 능력 부족이 아니라 PINN 자체의 한계다. 이 차이를 인지하고 시작하면 좌절이 줄어든다.</p>
  </div>
</div>

**PINN이 학습되지 않는 흔한 원인.**

첫째, **손실 항 사이의 스케일 불균형**. data loss, PDE residual loss, boundary loss의 절대값이 크게 다르면, 가장 큰 손실만 줄이는 방향으로 학습된다. 예를 들어, PDE residual이 100, boundary loss가 0.01이면 모델은 boundary를 거의 무시한다. 해결: 각 손실의 가중치를 조정하거나, 적응형 가중치(NTK-based, gradient-based) 사용. 또는 학습 전에 각 손실 항을 정규화하여 비슷한 스케일로 만든다.

둘째, **PDE의 내재적 어려움**. 경계층(boundary layer), 충격파(shock), 불연속이 있는 PDE는 PINN으로 풀기가 매우 어렵다. 신경망은 본질적으로 매끄러운 함수이므로, 급격한 변화를 잘 표현하지 못한다. 해결: 도메인 분할(domain decomposition), 적응형 샘플링, 특수 활성화 함수, 또는 PINN을 포기하고 FEM/FVM으로 돌아간다.

셋째, **잘못된 입력 정규화**. PDE가 다양한 스케일의 변수(예: x ∈ [0, 1m], t ∈ [0, 100s])를 가지면 신경망이 학습에 어려움을 겪는다. 해결: 모든 입력을 [-1, 1] 또는 [0, 1]로 정규화한 후, PDE도 정규화된 변수로 다시 작성한다. PDE의 변수 변환이 핵심이다.

넷째, **활성화 함수 선택**. ReLU는 PINN에 거의 적합하지 않다(그래디언트가 거의 항상 0 또는 상수이므로 PDE의 고차 미분이 잘 작동 안 함). PINN에는 tanh, sin, swish 같은 매끄러운 활성화 함수가 표준이다. 해결: tanh부터 시도하고, 필요하면 sin 또는 sin+tanh 조합.

다섯째, **샘플링 위치**. PINN의 학습 데이터는 collocation point다. 이 점들이 어디에 있느냐가 중요하다. 균등 격자 샘플링은 boundary나 변화가 큰 영역에서 부족할 수 있다. 해결: 적응형 샘플링(잔차가 큰 영역에 더 많은 점), Latin Hypercube Sampling, Sobol sequence.

**PINN 디버깅의 단계별 접근.**

1단계: **단순한 사례에서 시작**. 본인이 풀고 싶은 복잡한 문제 대신, 해석해가 있는 매우 단순한 사례(1D 정상 상태 열전도, 1D 파동방정식)에서 본인의 PINN 코드가 작동하는지 확인. 이것이 작동하지 않으면 본인의 코드에 버그가 있다. 작동한다면 코드는 정상이고, 문제는 본인의 복잡한 PDE의 어려움이다.

2단계: **개별 손실 항 시각화**. 학습 과정에서 data loss, PDE loss, boundary loss를 따로 그래프로 그린다. 어느 항이 줄지 않고 있는지 즉시 파악된다.

3단계: **예측과 잔차 시각화**. 학습 후 전체 도메인에 대한 예측을 그림으로 그린다. 잔차(PDE residual)도 시각화한다. 잔차가 큰 영역이 어디인지 보면 문제의 원인이 보인다.

4단계: **해석해 또는 FEM 결과와 비교**. 본인의 PINN 결과를 해석해(있다면) 또는 FEM 결과와 비교한다. 단순히 "PINN이 답을 냈다"는 충분하지 않다. 그 답이 옳은지 확인해야 한다.

5단계: **점진적 복잡화**. 1D 단순 사례에서 작동하면, 2D 단순 사례로, 그 다음 2D 복잡 사례로 단계적으로 확장. 한 단계씩 검증하면 어느 단계에서 실패하는지 보인다.

**PINN의 현실적 사용 권장.** 다음 경우에만 PINN을 시도한다. 첫째, **역문제(Inverse Problem)**: 측정 데이터로부터 미지 매개변수를 추정하는 문제. PINN은 forward 시뮬레이션에서는 종종 FEM보다 느리지만, 역문제에서는 강력한 강점을 가진다. 둘째, **데이터-물리 결합**: 일부 데이터가 있고 PDE 법칙도 있을 때. 데이터만 또는 PDE만으로는 해결되지 않는 문제. 셋째, **고차원 문제**: 5차원 이상의 PDE는 FEM이 차원의 저주에 걸리지만 PINN은 비교적 잘 작동한다. 넷째, **연속적 표현이 필요할 때**: 임의 점에서 평가가 필요한 응용. PINN은 연속 함수를 학습하므로 격자 외 점에서도 평가 가능.

다음 경우에는 PINN을 권하지 않는다. 첫째, 표준적인 forward 문제(예: 정상 상태 구조 해석). FEM이 훨씬 빠르고 정확. 둘째, 충격파, 불연속, 강한 비선형성이 있는 문제. 셋째, 산업 표준이 필요한 안전 critical 응용. PINN은 아직 표준화되어 있지 않다.

## PINN과 불확실성 정량화의 결합

ch17의 UQ 섹션과 ch24의 PINN을 결합하면 강력한 도구가 된다. 단일 PINN은 결정론적(deterministic) 예측을 한다. 즉, 같은 입력에 대해 항상 같은 출력. 하지만 현실에서는 측정 노이즈, 모델 불확실성, 매개변수 불확실성이 있다. 이 불확실성을 정량화한 PINN이 더 신뢰할 수 있는 결과를 낸다.

**Bayesian PINN (B-PINN).** 신경망의 가중치를 점 추정 대신 확률 분포로 다룬다. 같은 입력에 대해 여러 다른 가중치 샘플로 여러 예측을 하고, 그 분포가 불확실성이 된다. 구현 방법:

첫째, **Variational Inference**: 가중치의 사후 분포를 정규분포로 근사. 학습 시 정규분포의 평균과 분산을 학습. PyTorch의 `pyro` 또는 `bayesian-torch` 라이브러리.

둘째, **Monte Carlo Dropout**: 학습된 모델을 추론 시에도 dropout을 켠 채로 여러 번 실행. 매번 다른 예측을 얻고, 그 분산이 불확실성. 가장 단순한 방법.

셋째, **Deep Ensembles**: 같은 PINN을 다른 random seed로 5-10번 학습. 여러 모델의 예측 평균이 점 추정, 분산이 불확실성. 가장 신뢰할 수 있지만 학습 비용이 N배.

**불확실성의 두 가지 종류.** **Aleatoric Uncertainty(데이터 불확실성)**: 측정 노이즈에서 오는 본질적 불확실성. 데이터를 더 많이 모아도 줄어들지 않는다. **Epistemic Uncertainty(모델 불확실성)**: 모델의 한계에서 오는 불확실성. 더 많은 데이터로 줄일 수 있다. PINN의 외삽 영역에서 epistemic uncertainty가 크게 나타난다. 이것은 "이 영역에서는 모델을 신뢰하지 마세요"라는 경고가 된다.

**PINN+UQ의 활용.** 첫째, **신뢰 구간이 있는 예측**. "응력은 250±15 MPa"처럼 구간 추정. 의사 결정에 훨씬 유용. 둘째, **Out-of-distribution 탐지**. 본인의 데이터가 PINN의 학습 도메인 밖이면 epistemic uncertainty가 급증. 자동으로 외삽 위험을 알린다. 셋째, **Active Learning과의 결합**. 불확실성이 큰 영역에 추가 데이터를 모으면 모델 개선에 가장 효율적.

**한계와 비용.** Bayesian PINN은 일반 PINN보다 학습이 5-10배 느리다. 메모리도 더 많이 든다. 본인의 박사 연구가 안전 critical 응용이거나 의사 결정 지원이라면 이 비용을 감수할 가치가 있다. 단순한 forward 시뮬레이션이라면 일반 PINN으로 충분하다.

## 다물리 PINN과 결합 시뮬레이션

ch17에서 다물리 결합 시뮬레이션을 다뤘다. PINN도 다물리 문제에 적용할 수 있고, 일부 측면에서 전통 방법보다 우위가 있다. 하지만 어려움도 더 크다.

**다물리 PINN의 어려움.** 첫째, **여러 PDE의 동시 학습**. 열-구조 결합이라면 열전도 방정식과 탄성 방정식을 동시에 학습. 손실 항이 두 배로 늘어 가중치 조정이 더 어려움. 둘째, **물리 사이의 시간 스케일 차이**. 열은 느리게, 구조는 빠르게 반응. 학습 시 어느 스케일에 맞출지 결정. 셋째, **Coupling 항의 정확도**. 두 물리 사이의 상호작용 항이 작으면 PINN이 무시하기 쉽다.

**다물리 PINN의 두 가지 접근.**

첫째, **단일 네트워크**: 하나의 신경망이 모든 물리 변수(예: 온도와 변위)를 동시에 출력. 단순하지만 손실 균형이 어렵다.

둘째, **다중 네트워크**: 각 물리에 별도 네트워크. 네트워크들이 입력을 공유하거나 출력을 교환한다. 더 유연하지만 복잡하다.

**Sequential vs Coupled 학습.** 다물리 PINN을 학습할 때 두 가지 전략이 있다.

전략 A (Sequential): 한 물리를 먼저 학습한 후, 그것을 고정하고 다른 물리를 학습. 단순하지만 강한 결합에서는 정확도가 떨어진다. 약결합 문제에 적합.

전략 B (Coupled): 모든 물리를 동시에 학습. 더 정확하지만 학습이 어렵다. 강결합 문제에 적합.

**현실적 추천.** 다물리 PINN은 아직 활발한 연구 영역이다. 본인의 박사 연구가 다물리 결합 문제라면 PINN보다 전통 결합 시뮬레이션(ch17의 다물리 결합)을 우선 검토한다. 다물리 PINN은 매우 어려운 도전이고, 1년의 박사 시간을 PINN에만 쓰다가 결과를 얻지 못할 위험이 있다. 다만 본인의 박사 후반에 충분한 PINN 경험이 쌓이면 다물리 PINN을 시도할 수 있다.

## 하이브리드 모델링: 물리와 데이터의 균형

ch19에서 실험-시뮬레이션 융합을 다뤘고, ch24의 PINN은 본질적으로 물리(PDE)와 데이터의 결합이다. 하이브리드 모델링은 더 넓은 개념으로, 물리 모델과 데이터 모델을 다양한 방식으로 결합하는 모든 접근을 포함한다.

**하이브리드 모델링의 4가지 패턴.**

첫째, **Physics-Constrained ML**: ML 모델의 출력에 물리적 제약을 가한다. 예: 신경망의 마지막 층에 "예측이 양수여야 한다"는 제약을 부여(softplus 활성화). 또는 손실 함수에 물리 위반 패널티 추가. PINN이 이 패턴의 한 형태다.

둘째, **Physics-Informed Features**: 입력 특성에 물리적 변환을 추가. 예: 단순 변수(P, V, T) 대신 무차원 수(Reynolds, Prandtl)를 입력으로 사용. 도메인 지식을 모델에 주입.

셋째, **Hybrid Architecture**: 모델 아키텍처에 물리적 구조를 반영. 예: Hamiltonian Neural Networks(에너지 보존), Equivariant Neural Networks(대칭성). 본인의 시스템의 물리적 특성이 모델 구조에 반영된다.

넷째, **Residual Modeling**: 물리 모델로 1차 근사를 얻고, ML로 잔차(physics model이 못 잡는 부분)를 학습. 예: 단순 1D 모델의 결과 + ML이 학습한 보정. 가장 실용적이고 안정적인 패턴.

**Residual Modeling의 강점.** Residual modeling은 가장 보수적이지만 가장 안정적이다. 물리 모델의 신뢰성을 유지하면서 ML로 정확도를 보강한다. 이점: 첫째, **물리 일관성**. 물리 모델이 보존 법칙을 만족하면 전체 모델도 만족. 둘째, **데이터 효율**. ML은 작은 보정만 학습하면 되므로 적은 데이터로 충분. 셋째, **외삽 안정성**. 물리 모델이 외삽에서도 어느 정도 작동하면, 전체 모델도 외삽에서 안정적. 넷째, **해석 가능성**. 무엇이 물리이고 무엇이 ML 보정인지 명확히 분리.

**박사 연구에서 하이브리드 모델링의 가치.** 본인의 박사 연구가 "ML 모델 vs 물리 모델"의 단순 비교가 아니라, 두 가지를 결합한 새로운 모델을 제안한다면 그 자체가 큰 학술적 기여다. 공학 분야의 ML 논문 중에서 하이브리드 모델링은 가장 인용을 잘 받는 카테고리 중 하나다. 본인의 도메인 지식과 ML 기술을 모두 활용할 수 있는 영역이기 때문이다.

**현실적 시작점.** 본인이 처음 하이브리드 모델링을 시도한다면 다음 순서로 한다. 첫째, **Physics-Informed Features부터 시작**. 본인의 입력 특성에 물리적으로 의미 있는 변환을 추가. 가장 단순하고 즉시 효과적. 둘째, **Residual Modeling 시도**. 본인의 단순 물리 모델 + ML 잔차. 셋째, **Physics-Constrained Loss 추가**. 본인의 ML 모델 손실에 물리 위반 패널티 추가. 넷째, **PINN 또는 Hybrid Architecture**: 가장 어렵고 마지막 단계.

## PINN과 대리 모델의 비교 정리

본인이 박사 연구에서 PINN과 대리 모델 중 어느 것을 사용할지 결정할 때 도움이 되는 비교다.

| 측면 | PINN | 대리 모델(GP/NN) |
|------|------|------------------|
| 학습 데이터 필요 | 적음 (PDE가 정보 제공) | 많음 (학습 데이터에 의존) |
| PDE 지식 필요 | 필수 | 불필요 |
| 학습 속도 | 느림 | 빠름 |
| 추론 속도 | 빠름 | 빠름 |
| 정확도 (forward) | 종종 FEM보다 낮음 | 학습 데이터에 의존 |
| 외삽 가능성 | 제한적 | 매우 제한적 |
| 역문제 | 매우 강력 | 약함 |
| 다물리 | 어려움 | 가능 |
| 불확실성 정량화 | Bayesian PINN으로 가능 | GP는 자체 제공 |
| 산업 표준 | 아님 | 일부 영역에서 표준 |

**언제 PINN을 선택하는가.** PDE가 명확히 정의되어 있고, 데이터가 적고, 역문제이거나 데이터-PDE 결합 문제일 때.

**언제 대리 모델을 선택하는가.** PDE가 모호하거나, 입력-출력 매핑이 명확하고 데이터가 많을 때, 빠른 의사 결정 지원이 필요할 때, 베이지안 최적화와 결합할 때.

**둘을 결합할 수도 있다.** 가장 강력한 접근은 두 가지를 함께 사용하는 것이다. PINN으로 forward 시뮬레이션 데이터를 생성하고, 그 데이터로 대리 모델(GP)을 학습하여 빠른 추론을 가능하게 한다. 이 결합은 PINN의 물리 일관성 + 대리 모델의 빠른 추론 + GP의 불확실성 정량화를 모두 활용한다.

본인의 박사 연구가 컴퓨테이셔널 물리/공학 분야라면, PINN과 대리 모델을 모두 도구 상자에 갖추는 것이 가치 있다. 어느 것이 항상 최선이 아니라, 본인이 풀려는 문제에 따라 적절한 도구를 선택하는 능력이 박사의 역량이다.

## Neural Operator — PINN 이후의 새로운 패러다임

PINN과 전통 대리 모델을 다뤘다면, 최근(2020년 이후) 빠르게 성장하는 세 번째 패러다임이 있다. **Neural Operator**다. 이름이 낯설 수 있지만, 원리는 매우 강력하고 공학 연구에서 앞으로 몇 년간 핵심 도구가 될 가능성이 높다. 본인이 PDE 기반 연구를 한다면 이 개념을 반드시 알아야 한다.

<div class="highlight-box highlight-info">

**핵심 아이디어: 해를 배우는 게 아니라 연산자(operator)를 배운다.** PINN은 하나의 특정 PDE 인스턴스(예: 특정 경계 조건, 특정 파라미터)에 대한 해를 학습한다. 따라서 경계 조건이나 파라미터가 바뀌면 다시 학습해야 한다. Neural Operator는 PDE의 입력 함수(초기 조건, 경계 조건, 소스 항)에서 출력 함수(해)로 가는 **매핑 자체**를 학습한다. 한 번 학습하면 수많은 다른 입력에 대해 즉시 추론할 수 있다. 이것이 패러다임의 차이다.

</div>

**주요 Neural Operator 아키텍처.**

첫째, **DeepONet (Deep Operator Network, 2021)**. Lu et al.이 제안. 두 개의 서브네트워크(branch와 trunk)로 구성된다. Branch 네트워크는 입력 함수를 인코딩하고, trunk 네트워크는 출력 위치를 인코딩한다. 두 출력을 내적하여 해의 값을 얻는다. 이론적 보편 근사 정리(universal approximation theorem)로 뒷받침되는 방법이다. 비교적 단순한 구조이고 구현이 직관적이다.

둘째, **FNO (Fourier Neural Operator, 2021)**. Li et al.이 제안. 핵심 아이디어는 해 함수의 주파수 영역에서 선형 변환을 학습하는 것이다. FFT를 사용하여 입력을 주파수 공간으로 보내고, 학습 가능한 가중치로 저주파 성분을 변환한 후, 역 FFT로 돌아온다. 이 과정을 여러 층 쌓는다. **메쉬 독립적**이라는 놀라운 성질을 가진다. 학습 때 본 해상도와 다른 해상도에서도 추론이 작동한다. 유체역학(Navier-Stokes)에서 특히 강력하다.

셋째, **GraphNO (Graph Neural Operator)**. 불규칙 메쉬에 적용 가능하도록 FNO를 일반화. 복잡한 기하학이나 유한요소 메쉬에 적용할 때 유용하다. 구조해석, 지반공학 같은 불규칙 메쉬 분야에서 관심이 높아지고 있다.

넷째, **Transformer-based Operator**. 최근에는 Transformer 아키텍처를 Neural Operator로 사용하는 연구가 활발하다. Galerkin Transformer, OFormer 등. Attention 메커니즘의 장거리 의존성 모델링 능력을 활용한다.

**Neural Operator와 PINN의 핵심 차이 정리.**

| 측면 | PINN | Neural Operator |
|------|------|-----------------|
| 학습 대상 | 특정 문제의 해 | 문제 → 해 매핑 |
| 학습 데이터 | PDE 잔차 (자동 미분) | 다수의 (입력, 출력) 쌍 |
| 새 조건 대응 | 재학습 필요 | 즉시 추론 |
| 학습 비용 | 문제당 시간 | 한 번의 대량 학습 |
| 추론 비용 | 매우 낮음 | 매우 낮음 |
| 정확도 | 문제에 따라 | 보통 FEM보다 낮지만 충분 |
| 필요한 것 | PDE 명시 | 학습 데이터 (수천~수만 인스턴스) |

**Neural Operator가 빛나는 시나리오.**

1. **설계 최적화**: 수많은 다른 기하학적 구성을 평가해야 하는 경우. FEM을 매번 돌리면 수 주 걸릴 일이 학습된 Neural Operator로는 몇 초면 끝난다. 구조 최적화, 공기역학 형상 최적화 등.

2. **불확실성 전파**: 입력 파라미터의 분포를 출력 분포로 변환해야 하는 UQ 연구. 몬테카를로 샘플링 시 FEM 대신 Neural Operator를 사용하면 수천-수만 배 빨라진다.

3. **역문제**: 관측 데이터에서 숨겨진 파라미터(예: 지반 물성, 재료 결함 위치)를 추정하는 문제. 매 반복마다 forward 시뮬레이션이 필요한데, Neural Operator가 이를 저렴하게 만든다.

4. **실시간 제어**: 공정 제어, 디지털 트윈에서 실시간으로 PDE 해가 필요한 경우. FEM은 너무 느려서 불가능하지만 Neural Operator는 가능하다.

5. **기후/날씨 모델링**: FNO가 수치 예보 모델(ECMWF-IFS)과 비교할 만한 성능을 내면서 수천 배 빠르다는 것이 2022-2023년 증명되었다. 공학 분야도 유사한 혁명이 진행 중이다.

**Neural Operator의 현실적 한계.**

첫째, **대량의 학습 데이터 필요**. PINN이 "PDE만 알면 된다"라면, Neural Operator는 "수천 개의 (입력, 출력) 쌍이 필요하다". 이 데이터를 만들려면 기존 FEM/CFD 솔버로 수천 번 시뮬레이션을 돌려야 한다. 사전 데이터 생성 비용이 상당하다.

둘째, **외삽 약함**. 학습 데이터 분포를 벗어난 입력에서는 성능이 크게 떨어진다. 학습 때 본 레이놀즈 수 범위를 벗어나면 FNO가 실패한다. 이 한계는 현재 활발한 연구 주제다.

셋째, **물리 제약 보장 없음**. PINN은 설계상 PDE를 (근사적으로) 만족시킨다. Neural Operator는 데이터에 의존하므로, 물리 보존 법칙(질량, 에너지 보존)을 자동으로 만족시키지 않는다. 이를 보완하기 위해 Physics-Constrained Neural Operator 같은 변형이 연구되고 있다.

넷째, **복잡한 기하학**. FNO는 규칙적 격자에 가장 잘 작동한다. 복잡한 경계를 가진 실제 공학 문제에는 추가 기법이 필요하다.

**PINN에서 Neural Operator로 확장하기.** 본인이 이미 PINN 연구를 하고 있다면, Neural Operator로 자연스럽게 확장할 수 있다. 두 가지 경로가 있다.

경로 1: **PINN으로 데이터 생성 → Neural Operator 학습**. 본인의 PINN으로 다양한 입력에 대한 해를 생성하고, 이를 Neural Operator의 학습 데이터로 사용한다. 결과적으로 PINN의 정확도와 Neural Operator의 속도를 결합할 수 있다.

경로 2: **Physics-Informed Neural Operator (PINO)**. PINN의 물리 제약과 Neural Operator의 매핑 학습을 결합한 하이브리드. 학습 손실 함수에 데이터 손실 + PDE 잔차 손실을 모두 넣는다. 2022년 이후 활발히 연구되는 방향이다.

**구현 리소스.**

- **neuraloperator** (GitHub: neuraloperator/neuraloperator): FNO와 관련 방법의 공식 PyTorch 구현. 튜토리얼과 벤치마크 포함.
- **DeepXDE** (ch24에서 이미 언급): 최근 버전은 DeepONet도 지원.
- **Modulus** (NVIDIA): FNO, DeepONet, PINN을 모두 포함하는 상용급 프레임워크. HPC 환경에서 강력.
- **JAX 기반 구현**: 연구 코드는 JAX로 작성되는 경우가 많다. 자동 미분과 속도 모두 이점이 있다.

**연구 주제로서의 매력.** Neural Operator는 2021년 이후 NeurIPS, ICML, ICLR 같은 Top 학회에서 매년 수십 편의 논문이 나오는 활발한 분야다. 공학 분야(유체, 구조, 재료)에서도 적용 논문이 빠르게 늘고 있다. 박사 주제로 삼기에 좋은 이유는 다음과 같다. 첫째, 아직 성숙하지 않은 분야라 기여할 여지가 많다. 둘째, 공학 분야에 적용할 때 분야 지식이 결정적이어서, 단순 ML 연구자와 차별화할 수 있다. 셋째, 실용성이 높아 산학 협력 기회도 많다.

**주의: 트렌드에 휩쓸리지 말 것.** 동시에 경고가 필요하다. Neural Operator가 "모든 것을 바꿀" 것처럼 보이는 하이프 사이클에 있다. 실제로는 전통 FEM/CFD가 여전히 정확도와 신뢰성 면에서 우위를 가지는 영역이 많다. 본인의 연구 문제가 정말 Neural Operator를 필요로 하는지, 아니면 트렌드를 좇는 것인지를 냉정히 판단해야 한다. ch21의 강한 베이스라인 원칙이 여기서도 적용된다. 본인의 Neural Operator를 평가할 때 전통 FEM/CFD를 베이스라인으로 포함하고 정직하게 비교해야 한다. "10초 대 10시간" 같은 속도 비교는 흥미롭지만, 정확도 손실이 얼마나 되는지가 핵심이다.

> Neural Operator는 PINN 이후 물리-ML 융합 연구의 다음 물결이다. 본인이 컴퓨테이셔널 물리/공학 박사 과정이라면, 이 개념을 2026년 기준 필수 지식으로 간주하고 최소한 한 편의 FNO 또는 DeepONet 논문을 깊이 읽어 두는 것이 좋다. 본인의 박사 주제 자체가 Neural Operator가 아니더라도, 이 흐름을 이해하지 못하면 본인 분야의 최근 연구를 읽기 어려워진다.

## 역문제와 파라미터 추정 — PINN이 진짜 빛나는 영역

PINN의 가장 강력한 응용은 보통의 forward 시뮬레이션이 아니라 **역문제(inverse problem)**다. 역문제는 "관측 데이터로부터 모델의 숨겨진 파라미터나 초기/경계 조건을 추정하는" 문제다. 전통 FEM/FDM은 forward 문제(주어진 조건 → 해)에 최적화되어 있지만, 역문제에는 본질적으로 적합하지 않다. PINN은 이 영역에서 FEM을 명확히 압도하며, 많은 박사 연구의 실제 가치를 창출한다. 그런데 이것을 제대로 이해하고 활용하는 학생이 드물다.

<div class="highlight-box highlight-important">

**PINN의 "진짜 사용처"**. 많은 초심자가 "PINN으로 1D 열 방정식을 풀었다"는 forward 연구로 시작한다. 이것은 FEM으로 0.01초에 풀리는 문제를 PINN으로 30분 걸려 푸는 것이다. 학술적으로는 가치가 낮다. PINN의 진짜 가치는 역문제에 있다. 예: "이 열전도 측정 데이터로부터 재료의 열전도 계수를 추정하라." 이 문제에서 PINN은 FEM보다 훨씬 자연스럽고 강력하다. 본인의 PINN 연구 방향을 forward에서 inverse로 바꾸는 순간, 연구의 가치가 크게 올라간다.

</div>

**역문제의 6가지 유형.**

공학에서 자주 나타나는 역문제의 유형.

**1. 파라미터 추정 (Parameter Identification).**
시스템의 지배 방정식은 알지만 내부 파라미터(재료 상수, 열전도 계수, 점도 등)가 미지.

**예**:
- 콘크리트 구조물의 탄성 계수 추정 (변위 측정으로부터)
- 유체의 점도 추정 (속도 측정으로부터)
- 생체 조직의 탄성 특성 추정 (의료 영상으로부터)

**2. 초기 조건 추정 (Initial Condition Recovery).**
시스템의 현재 상태를 관측했을 때, 이전의 상태를 역추적.

**예**:
- 오염 물질의 발생원 위치 추적 (확산 후의 측정으로부터)
- 지진의 진원 위치 추정 (지표면 흔들림으로부터)
- 열원의 위치와 강도 추정

**3. 경계 조건 추정 (Boundary Condition Inference).**
내부 관측만 있고 경계 조건은 모름.

**예**:
- 바다의 해류 추정 (관측 부표들의 움직임으로부터)
- 대기 오염원 분포 (원격 측정으로부터)
- 엔진 내부 온도 분포 (외벽 온도만 측정)

**4. 장(Field) 재구성.**
드문드문한 측정으로부터 전체 장의 분포를 복원.

**예**:
- 혈류 속도 전체 장 (MRI 몇 개 슬라이스로부터)
- 구조물의 응력 분포 (몇 개 지점의 strain gauge로부터)
- 온도 분포 (몇 개 온도계로부터)

**5. 형상 추정 (Shape Identification).**
내부 결함, 공동, 경계의 형태를 추정.

**예**:
- 비파괴 검사에서 균열 위치와 크기
- 지하 동굴이나 지하수원 탐지
- 의료 영상에서 종양의 경계

**6. 모델 선택 (Model Identification).**
데이터로부터 적절한 지배 방정식 자체를 선택.

**예**:
- 재료의 구성 모델 (linear vs nonlinear, viscous vs plastic)
- 난류 모델 (k-ε vs k-ω vs LES)
- 반응 속도 모델

**왜 PINN이 역문제에 유리한가.**

전통 방법과 비교해보자.

**전통 접근**: "반복 forward 해결 + 최적화".
```
파라미터 추정 문제:
1. 파라미터 후보값 θ₀ 선택
2. θ₀로 FEM 시뮬레이션 실행
3. FEM 결과와 관측 데이터 비교 → 오차 계산
4. 오차를 줄이는 방향으로 θ 업데이트
5. 1-4 반복 (수백~수천 번)
```

이 접근의 문제:
- 매 반복마다 전체 FEM 실행 (비용 큼)
- 그래디언트를 수치적으로 추정해야 함 (느리고 부정확)
- 비선형 문제에서 지역 최적해에 빠짐
- 측정 노이즈를 자연스럽게 다루기 어려움

**PINN 접근**: "한 번의 통합된 학습".
```
파라미터 추정 문제:
1. PINN의 손실 함수에 두 항 포함:
   - PDE 잔차 (모든 격자점에서)
   - 관측 데이터 fitting (측정 지점에서)
2. θ를 학습 가능한 변수로 설정
3. 한 번의 학습으로 θ와 해 u(x,t)를 동시 최적화
```

PINN의 장점:
- 자동 미분으로 정확한 그래디언트
- 한 번의 학습으로 끝
- 관측 노이즈를 loss 함수에 자연스럽게 반영
- 관측 데이터가 드물 때도 PDE 잔차가 정규화 역할

**PINN 역문제 손실 함수의 구조.**

역문제 PINN의 손실은 기본 PINN보다 약간 복잡하다.

```python
def inverse_pinn_loss(model, x_data, u_data, x_pde):
    # 1. 데이터 피팅 항
    u_pred_data = model(x_data)
    data_loss = torch.mean((u_pred_data - u_data) ** 2)
    
    # 2. PDE 잔차 항 (파라미터 theta는 model.theta로 학습 가능)
    u_pred_pde = model(x_pde)
    du_dx = torch.autograd.grad(u_pred_pde, x_pde, 
                                 grad_outputs=torch.ones_like(u_pred_pde),
                                 create_graph=True)[0]
    d2u_dx2 = torch.autograd.grad(du_dx, x_pde,
                                   grad_outputs=torch.ones_like(du_dx),
                                   create_graph=True)[0]
    
    # PDE: d²u/dx² = f(u, theta)
    pde_residual = d2u_dx2 - model.theta * u_pred_pde
    pde_loss = torch.mean(pde_residual ** 2)
    
    # 3. 경계 조건 (있다면)
    bc_loss = ...
    
    # 가중합
    total_loss = lambda_data * data_loss + lambda_pde * pde_loss + lambda_bc * bc_loss
    return total_loss
```

**핵심**: `model.theta`를 학습 가능한 파라미터로 선언한다. PyTorch에서는 `nn.Parameter`로, TensorFlow에서는 `tf.Variable`로.

```python
class InverseModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(2, 64), nn.Tanh(),
            nn.Linear(64, 64), nn.Tanh(),
            nn.Linear(64, 1)
        )
        # 학습 가능한 물리 파라미터
        self.theta = nn.Parameter(torch.tensor([1.0]))
    
    def forward(self, x):
        return self.net(x)
```

학습 루프에서 `optimizer`에 `model.parameters()`를 넘기면 자동으로 `theta`도 함께 학습된다.

**가중치 균형의 문제.**

역문제 PINN의 가장 어려운 부분은 `lambda_data`, `lambda_pde`, `lambda_bc`의 균형을 맞추는 것이다. 잘못된 균형은 다음 문제를 만든다.

- `lambda_data`가 너무 크면: 측정에 과적합. 물리 법칙 무시.
- `lambda_pde`가 너무 크면: 물리 법칙은 만족하지만 파라미터 추정이 부정확.
- `lambda_bc`가 너무 크면: 경계에 과적합. 내부 해가 부정확.

**균형 찾기 전략**:

1. **그래디언트 기반 자동 가중치**: 각 loss 항의 그래디언트 크기를 맞춘다. `lambda`를 학습 중 동적으로 조정. Wang et al. (2021)의 "Learning the PINN loss landscape".

2. **불확실성 기반 가중치**: 각 loss를 "측정 불확실성"으로 정규화. Bayesian 관점.

3. **어닐링(Annealing)**: 학습 초기에는 `lambda_data`가 크게(데이터 따라가기), 나중에 `lambda_pde`가 커지는 스케줄.

4. **수동 튜닝**: 경험적으로 `lambda_pde / lambda_data` 비율을 log scale로 탐색. 보통 1/100, 1/10, 1, 10, 100.

초보자는 수동 튜닝부터 시작하고, 안 되면 자동 가중치로 넘어간다.

**데이터 양과 분포의 요구사항.**

역문제 PINN은 얼마나 많은 측정 데이터가 필요한가?

**최소 데이터 원칙**: 파라미터 수 × 10 정도의 측정 지점이 최소. 파라미터가 1개이면 10개 지점, 10개이면 100개 지점.

**데이터 분포의 중요성**: 측정 지점이 공간/시간적으로 고루 분포해야 한다. 한 지점에 몰려 있으면 정보가 부족하다.

**정보량 측정**: Fisher Information Matrix로 "어느 측정이 가장 정보적인가"를 정량화할 수 있다. 이것이 Optimal Experimental Design의 주제다.

**노이즈 대처**: 관측 데이터에 노이즈가 있으면 `lambda_data`를 낮춘다. "데이터를 완벽히 맞추지 말고, 노이즈 수준까지만 맞춰라"라는 원칙.

**역문제 PINN의 검증.**

역문제는 forward보다 검증이 어렵다. "정답"이 없기 때문이다. 다음 검증 전략.

**검증 1: 합성 데이터로 시작**.
진짜 파라미터를 알고 있는 시뮬레이션 데이터로 먼저 검증한다. PINN이 이 파라미터를 복원할 수 있으면 방법이 작동한다는 증거.

**검증 2: 실제 데이터로 비교**.
본인의 방법이 합성에서 작동하면, 실제 데이터에 적용. 이전 연구나 실험에서의 "참값"과 비교.

**검증 3: 다양한 시작점**.
같은 데이터에 대해 파라미터 초기값을 다르게 하여 여러 번 학습. 결과가 일관되면 신뢰할 수 있다. 일관되지 않으면 불량조건(ill-conditioned).

**검증 4: Sensitivity Analysis**.
입력 데이터에 작은 변화를 주었을 때 추정된 파라미터가 얼마나 변하는지. 안정적이어야 한다.

**검증 5: Posterior Distribution (Bayesian PINN)**.
점 추정이 아니라 분포를 추정. 각 파라미터에 대한 신뢰 구간을 제공. MCMC나 Variational Inference로 구현.

**역문제 PINN의 실전 예시 — 재료 파라미터 추정.**

구체적인 시나리오를 본다.

**문제**: 변위 측정으로부터 재료의 Young's modulus E 추정.

**설정**:
- PDE: $\nabla \cdot (E \nabla u) = f$ (탄성 방정식)
- 측정: 여러 지점의 변위 $u_i$
- 미지수: 재료의 E(x,y) 분포

**접근**:
```python
class ElasticityInversePINN(nn.Module):
    def __init__(self):
        super().__init__()
        # 변위를 예측하는 네트워크
        self.u_net = MLP(2, 1)
        # E 분포를 예측하는 네트워크
        self.E_net = MLP(2, 1)
    
    def forward(self, x):
        u = self.u_net(x)
        E = self.E_net(x)
        return u, E

def loss(model, x_data, u_data, x_pde):
    # 데이터 피팅
    u_pred, _ = model(x_data)
    data_loss = ((u_pred - u_data) ** 2).mean()
    
    # PDE 잔차
    u_pred, E_pred = model(x_pde)
    # ... gradient computation ...
    pde_residual = div(E_pred * grad(u_pred)) - f
    pde_loss = (pde_residual ** 2).mean()
    
    return data_loss + lambda_pde * pde_loss
```

이 접근은 E를 하나의 상수가 아니라 **공간적 분포**로 추정한다. 비균질 재료나 결함 감지에 유용.

**역문제 PINN 논문 작성의 팁.**

이런 연구를 논문으로 쓸 때 특히 강조할 점.

1. **왜 역문제인가를 명확히**. "Forward 시뮬레이션의 비효율" 외에 "역문제 자체가 필요한 응용"을 제시.

2. **전통 방법과의 공정한 비교**. 반복 FEM + 최적화가 현재 표준. 본인의 PINN이 그것보다 (a) 빠른가, (b) 더 정확한가, (c) 노이즈에 더 강한가, (d) 드문 데이터에서 작동하는가.

3. **노이즈 감응성 분석**. 측정 노이즈가 0%, 1%, 5%, 10%일 때 각각의 성능. 이것이 실제 적용 가능성을 보여준다.

4. **드문 데이터 시나리오**. 측정 지점이 10개, 50개, 100개일 때. 드물수록 본인의 방법이 유리해야 한다.

5. **이상치와 결측**. 측정이 완벽하지 않을 때의 robustness.

6. **실제 응용 케이스**. 합성 데이터만이 아니라 실제 실험 데이터나 현장 데이터에서의 시연.

이 요소들이 포함된 역문제 PINN 논문은 리뷰어에게 훨씬 강한 인상을 준다.

**역문제 PINN의 한계.**

솔직하게 인정해야 할 한계.

1. **학습 시간**: 여전히 forward 방법의 10-100배 느리다. 실시간 역문제에는 아직 부적합.

2. **하이퍼파라미터 민감**: loss 가중치, 네트워크 구조 등에 매우 민감. 튜닝이 어렵다.

3. **고차원에서의 스케일링**: 3D나 시공간 문제에서 그래디언트 계산이 폭발.

4. **물리 지식 필요**: 무작정 PINN을 쓴다고 되지 않는다. 해당 분야의 물리 직관이 있어야 한다.

5. **비유일성 (Non-uniqueness)**: 역문제는 종종 유일해가 없다. 여러 파라미터 조합이 같은 관측을 만들 수 있다. 이 경우 정규화나 사전 지식이 필요.

이런 한계를 인정하고 연구 설계 시 고려해야 한다.

> 본인이 PINN을 연구 주제로 삼고 있다면, forward 문제에만 머물지 말고 역문제로 확장하는 것을 적극 고려한다. 역문제 PINN은 단순한 "forward의 역"이 아니라 완전히 다른 연구 방향이며, 공학의 실제 응용에서 훨씬 큰 가치를 가진다. 박사 학위논문에 forward와 inverse 양쪽을 다룬다면 논문의 깊이와 폭이 크게 확장된다. 이것이 본인의 PINN 연구를 "또 하나의 forward 연구"에서 "의미 있는 과학적 기여"로 올려준다.
